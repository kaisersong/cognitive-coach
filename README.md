# cognitive-coach

`cognitive-coach` 是一个认知元能力教练型 skill。

它不直接给答案，而是通过渐进式提问，帮助用户把问题想深、想透、想完整。

**未发布。当前仅供作者自己测试、验证和迭代使用。**

## 一句话设计

先判断用户处在哪种思考状态，再只激活当前最缺的一个认知能力，而不是一上来把整套框架全部摊开。

## 渐进式披露结构

这个 skill 现在按三层组织：

### 第 1 层：快速路由

先判断用户属于哪一类入口：

- 带着已有分析来
- 从零开始梳理问题
- 明确说“感觉哪里不对但说不清”

这一层只决定第一个问题，不展开整套方法论。

### 第 2 层：核心循环

进入对话后，只执行一个最小闭环：

1. 提一个问题
2. 听用户回答
3. 判断哪个能力还没被激活
4. 再问下一个问题

这一层是 skill 的主工作面。

### 第 3 层：详细诊断

只有在需要更精细判断时，才展开：

- 九种认知元能力
- 常见盲点映射
- 高阶组合能力
- 结束信号与总结策略

这一层更像附录，而不是开场说明。

## 何时升级披露

默认停留在前两层。只有满足下面任一条件时，才升级到第 3 层：

- 用户已经回答了至少一轮，但问题仍明显卡住
- 用户给出了成型分析，需要做精细诊断
- 用户的回答同时暴露出多个盲点，需要判断先打哪一个
- 用户明确要求你“检验分析”“挑战我的思路”“看看我漏了什么”

换句话说：

- 第 1 层决定从哪里开始
- 第 2 层推进对话
- 第 3 层只在你需要更高分辨率时才打开

## 目标

- 把思维教练从“信息过载式提示词”改成“阶段式激活”
- 降低模型开场负担
- 让第一问更稳定、更聚焦
- 保留完整诊断能力，但推迟到真正需要时再展开
- 把 skill 从“理论说明文”压缩成“可执行操作手册”

## 使用边界

- 这是思维教练，不是答案生成器
- 核心交付是问题，不是结论
- 优先帮助用户发现：
  - 隐藏假设
  - 问题表征偏差
  - 目标冲突
  - 说不清的直觉异常
